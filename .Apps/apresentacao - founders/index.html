Módulo 01 · Sprint 1–2

Pipeline Visual

Tela principal do cockpit. Kanban com 6 colunas fixas, otimizado para ação rápida. Não é um clone do Kommo — é enxuto e orientado a agentes de IA.

Novo
Lead entra pelo site ou WhatsApp. Luma inicia qualificação automática em até 30s.
Qualificando
Luma coletando dados. Vendedor pode acompanhar ou intervir.
Orçamento enviado
ORS gerado e enviado. Follow-up automático ativado.
Negociando
Lead respondeu. Handoff para humano com contexto completo.
Ganho
Venda fechada. Dados alimentam o ICP automaticamente.
Perdido
Entra em reativação automática: 30–60–90 dias.

Card do lead mostra: nome da empresa, score colorido (verde >70, amarelo 40-70, vermelho <40), tipo + quantidade, valor do orçamento, tempo na etapa, origem e preview da última mensagem.

Módulo 02 · Sprint 1–2

Luma v2 — Agente SDR

Responde leads no WhatsApp em menos de 30 segundos, coleta dados de qualificação em conversa natural e cria o card no pipeline com informações estruturadas.

⚠ Problemas identificados na v1: campos "None", erro exposto ao cliente, sem fallback, sem retry.

1

Webhook receptor

Recebe mensagem da Meta API. Valida payload, extrai sender_id e message_text.

2

Contexto da conversa

Busca últimas N mensagens + dados coletados + estágio atual da qualificação.

3

State machine

Claude API recebe prompt com estágio explícito: nome → tipo → quantidade → CNPJ. Sabe exatamente o que perguntar.

4

Retry + fallback

Timeout 10s. Retry 3× com backoff exponencial (1s, 2s, 4s). Se falhar: mensagem genérica, nunca expõe erro técnico.

5

Extração estruturada

Resposta da Claude inclui mensagem para o cliente + JSON com dados extraídos via tool_use.

Módulo 03 · Sprint 3–4

ORS — Orçamentos Nativos

Gera orçamentos dentro do cockpit, sem trocar de sistema. Dados já coletados pela Luma preenchem o formulário automaticamente.

1

Dados pré-preenchidos

Vendedor abre o módulo dentro do card. Tipo, quantidade e dados da empresa já estão lá.

2

Catálogo de produtos

Preços base por tipo e modelo. Sugestão automática baseada no tipo solicitado.

3

Cálculo automático

Preço unitário ajustado pela quantidade. Tabela de descontos por volume. Margem configurável.

4

PDF + envio integrado

Numeração ORS-NNN. 1 clique envia pelo WhatsApp com mensagem personalizada por IA.

5

Atualização automática

Lead move para "Orçamento enviado". Follow-up ativado. Zero interação manual.

Módulo 05 · Sprint 3–4

Enriquecimento CNPJ

Enriquece automaticamente os dados do lead com informações públicas da empresa — habilitando scoring melhor e personalização da abordagem.

1

Trigger: CNPJ coletado

Luma coleta CNPJ em conversa. Enriquecimento dispara em background (BullMQ).

2

BrasilAPI — gratuito

Razão social, CNAE primário, porte (ME/EPP/Demais), endereço. Cache 30 dias.

3

CNPJ.ws — freemium

Sócios e administradores. Identifica o decisor. Capital social.

4

Ajuste de score

Porte grande +20, porte médio +10. Vertical saúde/indústria +15. Múltiplas filiais +10.

5

Atualização do card

Dados visíveis no painel lateral. Decisor identificado como "contato recomendado".

Módulo 06 · Sprint 7–8

Prospecção Outbound + ICP

Transforma o cockpit de "gerenciador de inbound" em plataforma de revenue completa. ICP calculado automaticamente a partir dos clientes fechados.

Não construir
Banco de dados próprio de empresas. Isso é o produto da Econodata. Scraper de LinkedIn viola ToS.
Construir
Camada de orquestração que consome APIs de dados (Google Maps, BrasilAPI) e entrega listas ranqueadas.
1

ICP Agent

Analisa clientes "Ganho". Extrai: CNAE mais frequente, faixa de quantidade, ticket médio, região, porte. Recalcula a cada 30 dias ou 10 novas vendas.

2

Busca + enriquecimento

Google Maps API busca empresas da categoria na região. BrasilAPI/CNPJ.ws enriquece com dados cadastrais.

3

Score de fit

Cada empresa recebe score 0-100 de compatibilidade com o ICP. Lista ranqueada por fit.

4

Agente de outbound

Gera mensagem personalizada por empresa. Clínica médica → jalecos. Fábrica → uniforme industrial. Envia via template WhatsApp aprovado.

// 03 — Arquitetura

Stack tecnológica

Decisões técnicas com justificativa de negócio. Cada escolha tem uma razão clara — não é "usar o que está na moda".

Frontend
React + Tailwind

Pipeline visual com drag-and-drop. Componentização. Ecossistema maduro.

Backend
Node.js (Fastify)

Async nativo para webhooks WhatsApp. Mesmo ecossistema do frontend.

Banco de dados
PostgreSQL + Prisma

Relacional robusto. JSONB para dados flexíveis de leads. Type-safety.

Filas
BullMQ + Redis

Follow-ups agendados. Retry de mensagens. Enrichment assíncrono.

IA
Claude API (Sonnet)

Classificação de leads, geração de mensagens, extração estruturada, agente SDR.

Deploy
Railway + Supabase

Deploy simples. PostgreSQL gerenciado. Custo baixo para MVP.

Decisão arquitetural central

State machine determinística com IA em nós específicos — não multi-agente autônomo. A IA extrai e entende linguagem natural. Regras determinísticas decidem o próximo passo. Quando algo dá errado, o debug é trivial. O vendedor confia porque o sistema faz o que ele espera.
