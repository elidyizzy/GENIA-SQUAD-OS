## Story
**As a** sistema,
**I want** baixar e processar os arquivos CSV da PGFN semanalmente,
**so that** o Banco de Leads seja atualizado automaticamente com novos devedores sem duplicatas.

## Acceptance Criteria
1. Script `scripts/pgfn-sync/sync.ts` executa sem erros com CNPJs reais
2. Download do CSV da PGFN feito em streaming (sem carregar tudo na memória)
3. Registros com `valor_divida < PGFN_DIVIDA_MINIMA` são ignorados
4. CNPJ normalizado (somente 14 dígitos, sem máscara) antes do upsert
5. Classificação calculada: A (≥ R$3M) / B (R$1M–R$3M) / C (abaixo do mínimo mas acima do filtro)
6. UPSERT em batch de 500 registros via Supabase: novo → INSERT, valor mudou → UPDATE, igual → ignora
7. `sync_logs` registra: iniciado_em, concluido_em, status, qtd_novos, qtd_atualizados, qtd_ignorados
8. Falha no processamento de um arquivo não aborta os demais — erro registrado em `sync_logs.erro`
9. GitHub Actions workflow `.github/workflows/pgfn-sync.yml` executa toda segunda às 06h UTC
10. Execução manual via `workflow_dispatch` funciona pelo GitHub UI

## Dev Notes
**Dependências do script:**
```json
{
  "axios": "^1.x",
  "csv-parser": "^3.x",
  "@supabase/supabase-js": "^2.x",
  "tsx": "^4.x"
}
```

**Formato PGFN dados abertos:** Os arquivos são CSVs separados por `;` com encoding `latin-1` ou `utf-8`. Verificar encoding real ao testar. Colunas esperadas (validar ao implementar):
- `CNPJ_DO_DEVEDOR`, `NOME_DO_DEVEDOR`, `VALOR_CONSOLIDADO`, `UF_UNIDADE_RESPONSAVEL`

**Upsert Supabase:** `onConflict: 'cnpj'` usa o índice UNIQUE da tabela `leads`. Registros com mesmo CNPJ e mesmo valor são ignorados automaticamente se `ignoreDuplicates: false` (default retorna o row).

**Classificação:**
```ts
function classificar(valor: number): 'A' | 'B' | 'C' {
  if (valor >= 3_000_000) return 'A'
  if (valor >= 1_000_000) return 'B'
  return 'C'
}
```

**Supabase client no script:** usar `createClient` com `SUPABASE_URL` + `SUPABASE_SERVICE_KEY` (service role, bypass RLS).
